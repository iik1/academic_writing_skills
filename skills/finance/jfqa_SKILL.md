# Journal Writing Skill: Journal of Financial and Quantitative Analysis

## Journal Profile
- **ABS 2024 Rating:** 4
- **Field:** FINANCE
- **Publisher:** Cambridge University Press (on behalf of University of Washington Foster School of Business)
- **Scope:** Theoretical and empirical research in financial economics, covering corporate finance, investments, capital and security markets, and quantitative financial methods.
- **Guidelines URL:** https://jfqa.org/style-guide/
- **Field baseline:** finance-field-writing-style
- **Articles spot-checked:** 0 (full-text access unavailable; findings below are GUIDELINE ONLY unless marked ENFORCED from production-stage style guide)

## Hard Constraints (Author Guidelines)

> These override FIELD_SKILL where they conflict.

### Structure
Follows FIELD_SKILL norm for overall section sequence. One notable production requirement: inline comments in the manuscript indicating approximate placement of each figure and table are required.

### Length
No explicit word or page limit stated in the style guide.

### Abstract
**Unstructured prose, single paragraph, non-indented, maximum 100 words.** Appears on the title page alongside author affiliations and contact information.
**ENFORCED** — hard limit stated in the production style guide, and substantially shorter than the 150–250-word range described in FIELD_SKILL.

### Citations
Author-year parenthetical format, inline only (no footnote citations). Years enclosed in parentheses. **For papers with 3 or more authors: all authors named on the first citation; abbreviated to "et al." on all subsequent citations.** Multiple citations separated by commas within a single set of parentheses. When citing a specific location (page or equation), include the locator inside the parentheses.
**ENFORCED** — the first-citation-all-names rule is a specific requirement not addressed in FIELD_SKILL.

### Headings
**Roman numerals for main sections (I, II, III, …); letters for subsections (A, B, C, …); Arabic numerals for sub-subsections (1, 2, 3, …).** In-text cross-references use the full format, e.g., "Section III.B.2". All heading levels use title case.
**ENFORCED** — this is the dominant delta from FIELD_SKILL, which states that Arabic numerals throughout are standard and Roman numerals appear only rarely.

### Tables and Figures
**Titles and descriptions are positioned above the table or figure**, not below. Multiple panels within a single figure are called "graphs"; each requires an individual title. Column headers that are sequential numerals appear without parentheses (e.g., "4" not "(4)"). Files are submitted in editable (Excel-compatible) format.
**ENFORCED** — above-placement is explicit in the style guide; FIELD_SKILL does not specify placement direction.

### Explicit Prohibitions
- No footnote citations; all citations are inline.
- No hyphenation in any text (hyphenation must be disabled).
- Multiplication must be indicated with "×" not "·".

## Observed Deltas from FIELD_SKILL

> Only dimensions where this journal deviates from the field baseline.

### Structural Deviations
**Section heading numbering is Roman, not Arabic.** FIELD_SKILL states that Arabic numerals (1, 2, 3, 2.1, 2.2) are standard across leading finance journals and that Roman numerals appear only rarely. JFQA mandates Roman numerals for main sections and a distinct A/B/C lettering system for subsections. This is a consistent, enforced departure.
**Classification: GUIDELINE ONLY** (confirmed in style guide; article-level verification not possible due to access constraints; given this is the production editing guide, compliance in published articles should be high).

### Abstract Length
FIELD_SKILL describes the field norm as 150–250 words. JFQA imposes a hard 100-word ceiling. Authors must condense substantially relative to the field norm.
**Classification: ENFORCED**

### Citation Format (Multi-Author First Citation)
FIELD_SKILL describes author-year format but does not specify the treatment of first-citation multi-author works. JFQA explicitly requires that all authors be named on the first citation when there are three or more, with et al. used only from the second citation onward.
**Classification: GUIDELINE ONLY** (stated in production guide; not confirmed against articles)

### Table/Figure Title Placement
FIELD_SKILL does not specify whether titles appear above or below tables and figures. JFQA explicitly requires above-placement for both.
**Classification: GUIDELINE ONLY** (stated in production guide; not confirmed against articles)

### Other Journal-Specific Patterns
- **Oxford comma required:** A comma before "and" in lists of three or more items is mandatory.
- **Code sharing policy:** A separate JFQA Code Sharing Policy applies to accepted papers; compliance is a condition of publication, not merely encouraged.
- **Dual submission policy:** JFQA maintains an explicit dual-submission policy. Authors must confirm compliance at submission.

## Common Pitfalls
1. **Abstract over 100 words.** The 100-word ceiling is well below field norms; drafts written to a standard finance abstract length will need significant compression.
2. **Arabic section numbering.** Authors familiar with JF, JFE, or RFS (which use Arabic numerals) will need to reformat all headings to Roman/letter/numeral hierarchy (I → A → 1).
3. **Et al. on first citation.** Using et al. immediately for 3+ author papers — the field convention at many outlets — violates JFQA's requirement to name all authors on first citation.
4. **Table/figure titles below the exhibit.** The field default in many word-processor templates places titles below; JFQA requires them above.
